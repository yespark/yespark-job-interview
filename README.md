git clone git@github.com:yespark/yespark-job-test.git

### Explications

> L'application a été configuré de manière très minimaliste pour se concentrer sur la codebase lors de la session.

> L'idée de ce live coding n'est pas forcément d'arriver au résultat final absolument, mais surtout à voir votre méthodologie de penser, de raisonnement, de travail que ce soit pour réaliser une fonctionnalité ou bien résoudre un bug.

> Pour que cela soit réalisable, la plupart des fichiers ont déjà été crée.

> Facultative, mais il est apprécié si vous pouvez faire l'implementation selon le paradigme de Hotwire,
